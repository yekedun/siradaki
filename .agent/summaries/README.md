# Summaries

Use this structure:

- `sessions/` for small per-session summaries
- `weekly/` for weekly rollups
- `monthly/` for monthly rollups when the project grows

Rules:

- keep session summaries short
- write weekly rollups from sessions, not from memory
- avoid duplicate progress logs
- do not recreate a long-running `progress.md`
- add metadata headers to summaries for automation and retrieval
